# 폴더 패널 트리뷰 추가

- 상태: Done

## 요구사항

선택한 폴더 패널에서 메뉴 > 트리뷰 항목으로 트리뷰를 토글할 수 있어야 한다.
- 트리뷰는 선택한 폴더 패널 내부 왼쪽에 표시
- 트리뷰는 켠 시점의 폴더를 루트로 표시
- 트리뷰에 닫기(x) 버튼 제공
- 트리 하위 폴더를 선택할 때 경로 텍스트필드는 트리뷰를 연 시점의 루트 경로로 고정 표시

## 원인 분석 또는 설계

현재 `FolderBrowser`는 파일 목록, 미리보기, 메타정보만 제공하고 패널 내부 보조 탐색 UI가 없다. 경로 텍스트박스는 항상 `CurrentPath`를 그대로 반영하므로, 트리뷰 기반 하위 탐색을 추가하려면 트리 루트 표시 경로와 실제 현재 탐색 경로를 분리해서 다뤄야 한다.

설계 방향:
- 트리뷰 표시 여부는 `FolderBrowser` 인스턴스별 상태로 관리
- 트리뷰를 켜는 순간의 `CurrentPath`를 트리 루트 기준 경로로 저장
- 트리에서 하위 폴더 선택 시 실제 목록은 해당 하위 폴더로 이동하지만, PathBox는 루트 기준 경로를 유지
- 트리뷰가 열린 상태에서 일반 탐색(뒤로/앞으로/상위/목록 더블클릭/직접 경로 입력)으로 이동하면 새 현재 경로를 루트로 다시 고정

## 구현 내용

- `FolderBrowser` 본문 좌측에 접이식 트리뷰 영역을 추가하고 닫기(x) 버튼을 배치했다.
- 트리뷰는 패널별 독립 상태로 관리되며, 메인 메뉴 `보기 > 트리뷰`에서 활성 패널 기준으로 토글된다.
- 트리뷰를 켜는 순간의 `CurrentPath`를 트리 루트 경로로 저장하고 상단 텍스트에 표시한다.
- 트리에서 하위 폴더를 선택하면 파일 목록과 미리보기 대상은 해당 하위 폴더로 이동하지만, `PathBox`는 루트 경로를 유지하도록 분리했다.
- 트리뷰가 열린 상태에서 일반 탐색(경로 입력, 목록 더블클릭, 뒤로/앞으로/상위 이동 등)으로 이동하면 새 위치를 트리 루트로 다시 고정하고 트리를 재구성한다.
- 트리 항목은 하위 폴더를 지연 로딩해 초기 렌더링 부담을 줄였다.
- 트리뷰를 닫아도 좌측 컬럼 폭이 남아 빈 공간을 차지하는 문제를 확인했고, 컬럼 폭 자체를 0으로 전환하도록 보완 중이다.
